# 시스템 개요

## 1. 이 시스템은 뭐 하는 시스템인가요?

**미국 주식 자동매매 시스템**입니다.

세 가지 정보를 종합해서 "어떤 종목을 살지/팔지"를 판단하고, 한국투자증권(KIS) API를 통해 실제로 주문까지 자동으로 넣어줍니다.

| 정보 종류 | 출처 | 무엇을 보나 |
|---|---|---|
| **경제 지표** | FRED (미국 연방준비은행 데이터) | 금리, 인플레이션, 실업률 등 거시 환경 |
| **주가/지수** | Yahoo Finance | S&P500, 나스닥, 개별 종목 가격 |
| **뉴스 감성** | AlphaVantage | 종목별 뉴스가 긍정적인지 부정적인지 |
| **AI 검토** | Claude (Anthropic) | 매수 직전 마지막 한 번 더 검토 |

## 3. 시스템이 켜질 때 무슨 일이 일어나나요?

`python run.py` 실행 시 순서:

```
[1] uvicorn이 FastAPI 앱(app/main.py) 띄움
       ↓
[2] lifespan 시작 → startup() 함수 호출
       ↓
[3] update_economic_data_in_background() 실행
    → 어제까지의 경제/주가 데이터를 DB에 저장
       ↓
[4] start_scheduler()       → 자동 매수 스케줄러 ON (5분마다 체크)
[5] start_sell_scheduler()  → 자동 매도 스케줄러 ON (1분마다 체크)
       ↓
[6] 서버 대기 상태 (API 요청 + 백그라운드 스케줄러 동시 실행)
```

해당 코드: `app/main.py:14-51`

## 4. 자동매매는 언제·어떻게 동작하나요?

### 4-1. 자동 매수 (`scheduler.py:444`)

| 항목 | 내용 |
|---|---|
| **언제** | 미국 동부 시간(ET) **평일 10:30~10:35** (장 시작 1시간 후) |
| **체크 주기** | 5분마다 시간 체크, 조건 맞을 때 1회 실행 |
| **흐름** | 1) 보유 종목 조회 → 2) 추천 종목 받아오기 → 3) **Claude AI 최종 검토** → 4) 가용 자금의 10%씩 분산 매수 → 5) ATR 기반 익절가/손절가 계산 후 `trade_records`에 저장 |
| **중복 방지** | 같은 날 이미 매수했거나 보유 중인 종목은 스킵 |

### 4-2. 자동 매도 (`scheduler.py:272`)

| 항목 | 내용 |
|---|---|
| **언제** | 미국 장 시간(평일 9:30~16:00 ET) |
| **체크 주기** | **1분마다** 매도 조건 체크 |
| **매도 조건** | 익절가 도달 / 손절가 도달 / 신호 변화 등 |
| **부가 작업** | KIS 원장과 DB의 `trade_records` 정합성 자동 동기화 (`_reconcile_orders`) |

### 4-3. 경제 데이터 자동 수집 (`scheduler.py:705`)

| 항목 | 내용 |
|---|---|
| **언제** | 한국 시간 매일 **06:05** (미국 장 마감 후) |
| **하는 일** | FRED + Yahoo Finance에서 어제 데이터를 받아와 DB에 추가 |

## 5. 데이터베이스(Supabase) 테이블

| 테이블 | 역할 |
|---|---|
| `economic_and_stock_data` | **모든 데이터의 기반** — 날짜별 경제지표 + 주가 (60+ 컬럼) |
| `stock_analysis_results` | 종목별 ML 예측 결과 (상승확률·예측가) |
| `stock_recommendations` | 기술적 지표로 필터링한 추천 종목 |
| `ticker_sentiment_analysis` | 종목별 뉴스 감성 점수 |
| `trade_records` | 매수/매도 거래 내역 (status: buy_ordered → holding → sell_ordered → sold) |
| `llm_decision_logs` | Claude AI의 매수 BUY/HOLD 판정 기록 |
| `access_tokens` | KIS API 토큰 캐시 (만료 시간 포함) |
| `predicted_stocks` | ML 모델 예측 결과 보관 |

## 6. 외부 API 정리

| API | 용도 | 인증 방식 |
|---|---|---|
| **KIS (한국투자증권)** | 잔고 조회, 주문 실행, 현재가/일봉 조회 | OAuth 토큰 (24시간) |
| **FRED (St. Louis Fed)** | 미국 경제 지표 (금리, 실업률 등) | API Key |
| **Yahoo Finance Chart API** | 주가, 지수, 환율 | 인증 불필요 |
| **AlphaVantage** | 종목별 뉴스 + 감성 점수 | API Key |
| **Anthropic (Claude)** | 매수 후보 최종 검토 | API Key |
| **Supabase** | DB | URL + Key (현재 anon, RLS 비활성화 권장) |

## 7. 핵심 흐름 요약 (한 줄로)

> **새벽에 데이터 모으고 → 기술분석으로 후보 추리고 → AI가 한 번 더 거른 후 → 미국 장 열리면 사고 → 1분마다 팔 타이밍 보고 → 익절/손절가 닿으면 자동 매도**

## 8. 한국 시간 vs 미국 시간 정리

| 시간대 | 코드 위치 | 역할 |
|---|---|---|
| **한국(KST)** | `pytz.timezone('Asia/Seoul')` | 경제 데이터 수집 스케줄(06:05), 로그 표시 |
| **뉴욕(ET)** | `pytz.timezone('America/New_York')` | 매수(10:30), 매도(9:30~16:00), 거래 시간 판단 — **서머타임 자동 처리** |

미국 장 시간 한국시간 환산: 서머타임 적용 시 **22:30~05:00**, 미적용 시 **23:30~06:00**.
