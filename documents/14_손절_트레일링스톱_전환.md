# 14. 손절 로직 전환: 고정 ATR 익절/손절 → 트레일링 스톱(Chandelier Exit)

> 작성일: 2026-05-30
> 대상 코드: `app/utils/scheduler.py`, `app/services/stock_recommendation_service.py`, `app/services/notification_service.py`
> DB 변경: `trade_records.trail_high` 컬럼 추가

---

## 1. 배경 — 왜 바꿨나

실제 계좌 자동매매 1개월치(2026-05-04 ~ 2026-05-28, 14거래일) 결과를 분석한 결과,
**손절이 너무 자주 발생해 수익률을 깎아먹는** 문제가 확인됐다.

### 기존 로직 (변경 전)
- 매수 시점 ATR로 고정 청산선 설정
  - 익절가 = 매수가 + ATR × **2.5**
  - 손절가 = 매수가 − ATR × **1.5**  → 손익비(R:R) 고정 1.67 : 1
- 매도 폴링(1분 주기)에서 `현재가 ≥ 익절가`면 익절, `현재가 ≤ 손절가`면 손절

### 데이터로 드러난 문제
- 매도 14건 중 **손절 7건(50%)**, 평균 손절폭 **−5.26%**
- 손절은 전부 "손절선에 겨우 닿고 잘린" 휩쏘(whipsaw) 패턴 — 큰 손실은 없음
- 저변동성 대형주(COST·MDLZ 등)는 ATR이 작아 손절선이 **−3%대**에 형성 → 일반 노이즈에도 손절
- 수익을 낸 종목(MU +24.5%, INTC +15.5%, AMD +15.1%)은 전부 손절폭이 넓었던 고변동성 종목

## 2. 백테스트 근거

실제 진입 21건(PYPL 초기화 매도 제외)을 그대로 두고 **손절/익절 방식만 바꿔** 재현.
상승장(5월 실거래 구간) + 하락장(과거 4개 실제 하락 구간) 양쪽에서 검증.

| 전략 | 상승장 | 하락장(4구간) | 하락장 손절률 |
|---|---|---|---|
| 기존 1.5×/2.5× (고정) | +1.65% | −3.90% | 89% |
| 2.5×/3.0× (고정) | +1.61% | −4.07% | 68% |
| 3.0×/4.0× (고정) | +2.59% | −4.41% | 62% |
| Buy & Hold (무손절) | +4.03% | −1.02% | 0% |
| **🏆 트레일링 3.0×(무익절)** | **+3.74%** | **−3.78%** | 86% |
| 트레일링 3.0× + 고정익절 4.0× | +2.27% | −4.22% | 85% |

### 핵심 발견
1. **진짜 병목은 손절이 아니라 "고정 익절"이 승자를 일찍 잘라낸 것**이었다.
   트레일링에 고정 익절(TP 4.0×)을 붙이면 +3.74% → +2.27%로 오히려 하락.
   → **고정 익절을 제거**하고 승자를 트레일링 스톱이 청산할 때까지 보유해야 한다.
2. 트레일링 3.0×는 **상승장·하락장 양쪽 모두 기존 설정을 능가**.
   - 상승장: Buy&Hold(+4.03%)의 93%를 회수
   - 하락장: 기존(−3.90%)보다도 약간 나으면서 꼬리위험은 스톱으로 방어
3. 트레일링 2.5×는 너무 좁아 휩쏘 → 배수는 **3.0이 적정**.

### 한계 (반드시 인지)
- 표본이 작다(상승장 1구간 3주, 하락장 4구간). 1%p 미만 차이는 노이즈일 수 있음.
- 단 **"트레일링 3.0× > 기존"은 양방향·양국면에서 일관**되어 방향성은 신뢰 가능.
- 백테스트는 ATR 배리어만 재현(기술적/신호 매도, 수수료·슬리피지 미반영).
- 깊은 약세장(2025 −19%)에선 여전히 마이너스 — 만능 아님, 꼬리위험 완화일 뿐.

## 3. 변경 후 로직 (트레일링 스톱 / Chandelier Exit)

```
ATR_TRAIL_MULT = 3.0            # 단일 소스: stock_recommendation_service.py

매수 시:
  trail_high       = 매수가          # 고점 앵커 (진입가에서 출발)
  stop_loss_price  = 매수가 − ATR × 3.0
  take_profit_price = NULL          # 고정 익절 제거

매도 폴링(1분 주기)마다:
  new_high  = max(trail_high, 현재가)
  new_stop  = new_high − ATR × 3.0          # 고점이 올라야만 상향, 하락 시 고정(래칫)
  if new_high > trail_high:                 # 고점 갱신 시 DB 업데이트
      trail_high      = new_high
      stop_loss_price = new_stop
  if 현재가 ≤ new_stop:                       # 청산
      → 매도 (sell_reason = "trailing_stop")
```

- ATR은 매수 시점 값을 고정 사용(`trade_records.atr`).
- 손절선은 **고점을 따라 위로만** 움직이고 절대 내려가지 않는다(래칫).
- 익절 상한이 없으므로 추세가 지속되는 한 계속 보유 → 승자 추종.

## 5. DB 마이그레이션

`db_backup/schema/alter_trade_records_trail_high.sql` 실행 필요:

```sql
ALTER TABLE trade_records ADD COLUMN IF NOT EXISTS trail_high FLOAT8;

UPDATE trade_records
SET trail_high = buy_price
WHERE status IN ('holding', 'buy_ordered', 'sell_ordered')
  AND trail_high IS NULL
  AND buy_price IS NOT NULL;
```

- 컬럼 미생성 상태로 매수가 일어나면 insert 시 에러 → **반드시 배포 전 실행**.
- 기존 보유 종목은 위 UPDATE + 애플리케이션 백필 로직이 `trail_high`/손절선을 재설정한다.

## 6. 운영 노트 / 롤백

- **`sell_reason`에 `trailing_stop` 신규 값** 등장. 수익률 통계 집계 시 이 값을 포함할 것
  (트레일링 청산은 익절·손절 양쪽 모두 가능하므로 별도 분류).
- **튜닝**: 손절을 더/덜 민감하게 하려면 `ATR_TRAIL_MULT` 한 곳만 변경(2.5=더 타이트, 3.5=더 느슨).
- **검증 권장**: 실계좌 적용 전 `KIS_USE_MOCK=true` 모의계좌로 한동안 병행 관찰.
- **롤백**: 코드를 이전 커밋으로 되돌리면 됨. `trail_high` 컬럼은 남겨도 무해(기존 로직은 무시).
