# 지표 연산 및 종합 점수 계산 (코드베이스 정밀 해설)

> 이 문서는 매수 후보 종목의 **종합 점수(composite_score)** 가 실제로 어떻게 계산되는지를, 코드 라인 단위로 추적해 정리한 문서입니다.
> 모든 코드 인용은 `app/services/stock_recommendation_service.py` 와 `predict_colab.py` 에서 가져왔습니다.

---

## 0. 한 페이지 요약 (TL;DR)

매수 후보의 종합 점수는 다음 6개 정규화된 부분 점수의 **가중합**입니다.

```
composite_score =
    0.25 * rise_score        ← ML 상승확률 점수
  + 0.25 * tech_score        ← 기술적 지표(SMA·RSI·MACD) 점수
  + 0.20 * sentiment_score   ← AlphaVantage 뉴스 감성 점수 (정규화)
  + 0.15 * volume_score      ← 거래량 비율 점수
  + 0.10 * adx_score         ← ADX(추세 강도) 점수
  + 0.05 * vix_score         ← VIX(시장 공포) 점수
```

위치: `app/services/stock_recommendation_service.py:754-762`

이 값이 **0.3 이상**이고, 하드 블록(VIX>35, RSI>80) 등을 통과한 종목만 LLM 검토로 넘어갑니다.

## 1. 사용자가 알고 있던 가중치 검증

| 항목 | 사용자 인식 | 코드 실제 값 | 일치 여부 |
|---|---|---|---|
| ML 상승확률 | 25% | `0.25 * rise_score` | ✅ 일치 |
| 기술적 분석 | 25% | `0.25 * tech_score` | ✅ 일치 |
| 감성 점수 | 20% | `0.20 * sentiment_score` | ✅ 일치 |
| 거래량 | 15% | `0.15 * volume_score` | ✅ 일치 |
| ADX | 10% | `0.10 * adx_score` | ✅ 일치 |
| VIX | **−5% 감점** | **+5% 가중치, 그러나 vix_score는 0 또는 음수** | ⚠️ 결과적으로는 일치 |

### VIX의 진실 (중요)

코드 자체는 `0.05 * vix_score` 로 **양수 가중치**를 곱하지만, `vix_score` 가 다음과 같이 0 또는 음수만 갖도록 설계되어 있습니다.

```python
# stock_recommendation_service.py:744-752
if vix_value is None:
    vix_score = 0.0
elif vix_value < 20:
    vix_score = 0.0       # 평온장 → 영향 없음
elif vix_value < 30:
    vix_score = -0.2      # 약간 불안 → -0.2
else:
    vix_score = -0.5      # 공포 → -0.5
```

따라서 VIX의 실제 종합점수 기여도는 **`0.05 × (0 ~ -0.5) = 0 ~ -0.025`** 사이입니다.
즉, 사용자가 말한 "VIX는 −5%로 감점"은 **결과적으로는 정확**하지만, 정확한 표현은 **"VIX는 가중치 5%이고 점수가 0 또는 음수라 감점만 발생"** 입니다.

추가로, **VIX > 35이면 매수 전면 중단(하드 블록)** 이 들어가 있어서 실질적으로 VIX 영향은 두 단계입니다:
- VIX < 35: composite_score에서 -0.025 ~ 0의 감점
- VIX > 35: 그 종목 자체가 결과 리스트에 포함되지 않음 (`stock_recommendation_service.py:667-670`)

### 2-1. ML 상승확률 점수 (`rise_score`) — 가중치 25%

**원천 데이터**: `predict_colab.py` 의 Transformer 모델이 만든 `Rise Probability (%)`.

#### 모델 구조 (`predict_colab.py:130-163`)

```python
def transformer_encoder(inputs, num_heads, ff_dim, dropout=0.1):
    attention_output = MultiHeadAttention(num_heads=num_heads, key_dim=inputs.shape[-1])(inputs, inputs)
    attention_output = Dropout(dropout)(attention_output)
    attention_output = Add()([inputs, attention_output])
    attention_output = LayerNormalization(epsilon=1e-6)(attention_output)
    ffn = Dense(ff_dim, activation="relu")(attention_output)
    ffn = Dense(inputs.shape[-1])(ffn)
    ffn_output = Dropout(dropout)(ffn)
    ffn_output = Add()([attention_output, ffn_output])
    ffn_output = LayerNormalization(epsilon=1e-6)(ffn_output)
    return ffn_output
```

- **입력 1(stock_inputs)**: 27개 종목 가격 시퀀스 (LOOKBACK=90일)
- **입력 2(econ_inputs)**: 37개 경제 지표 시퀀스 (LOOKBACK=90일)
- 각 입력에 4-layer Transformer Encoder 적용 → Dense(64) → Add → Dense(128) → Dropout(0.2) → GlobalAveragePooling1D → Dense(27)
- **예측 시점**: `i + FORECAST_HORIZON - 1` (즉, 14일 후 가격)

#### 상승확률 산식 (`predict_colab.py:205-231`)


### 2-2. 기술적 지표 점수 (`tech_score`) — 가중치 25%

3개의 기술 지표(골든크로스, RSI, MACD)를 가중합한 후 0~1로 정규화합니다.

#### SMA (단순 이동평균) — `stock_recommendation_service.py:65-67`

```python
def calculate_sma(self, series, period):
    return series.rolling(window=period).mean()
```

- **SMA20** (20일), **SMA50** (50일) 두 개 계산
- **골든크로스**: `SMA20 > SMA50` 이면 True (단기선이 장기선 위, 상승 추세)

#### RSI (Wilder's Smoothing) — `stock_recommendation_service.py:73-97`

```python
def calculate_rsi(self, series, period=14):
    # 비거래일(가격 변동 0) 제거 - ffill로 채워진 중복일 처리
    trading_series = series[series.diff() != 0].copy()
    if len(series) > 0:
        trading_series = pd.concat([series.iloc[:1], trading_series]).drop_duplicates()
    if len(trading_series) < period + 1:
        return pd.Series([50] * len(series), index=series.index)

    delta = trading_series.diff().dropna()
    gain = delta.where(delta > 0, 0)
    loss = -delta.where(delta < 0, 0)
    # Wilder's Smoothing (alpha = 1/period)
    avg_gain = gain.ewm(alpha=1/period, min_periods=period, adjust=False).mean()
    avg_loss = loss.ewm(alpha=1/period, min_periods=period, adjust=False).mean()
    rs = avg_gain / avg_loss
    rsi = 100 - (100 / (1 + rs))
    rsi = rsi.reindex(series.index, method='ffill')
    return rsi
```

- **업계 표준 Wilder's Smoothing** (TradingView·Investing.com과 동일 방식)
- 주말/공휴일 ffill 데이터를 RSI에서 제거하기 위해 `diff() != 0` 필터링
- **`rsi_buy = rsi <= 65`** (정상 매수 구간), 65 초과는 과열로 간주
- **RSI > 80은 하드 블록** (매수 후보에서 무조건 제외, `stock_recommendation_service.py:677-679`)

#### MACD — `stock_recommendation_service.py:99-105`

```python
def calculate_macd(self, series, short_period=12, long_period=26, signal_period=9):
    short_ema = self.calculate_ema(series, short_period)
    long_ema = self.calculate_ema(series, long_period)
    macd = short_ema - long_ema
    signal = self.calculate_ema(macd, signal_period)

```

- 표준 12/26/9 설정

### 2-5. ADX (추세 강도) 점수 (`adx_score`) — 가중치 10%

**ADX (Average Directional Index)**: 추세의 강도를 측정 (방향과 무관). 14일 기준이 표준.

#### 계산 로직 (`stock_recommendation_service.py:153-225`)

```python
def calculate_adx(self, daily_data, period=14):
    if len(daily_data) < period * 2 + 1:
        return None
    data = list(reversed(daily_data))   # 최신일이 0번 → 시계열 정렬
    highs  = [float(d.get("high", "0") or "0") for d in data]
    lows   = [float(d.get("low", "0") or "0") for d in data]
    closes = [float(d.get("clos", "0") or "0") for d in data]

    # True Range, +DM, -DM 계산
    for i in range(1, len(data)):
        high_diff = highs[i] - highs[i - 1]
        low_diff  = lows[i - 1] - lows[i]
        tr = max(highs[i] - lows[i], abs(highs[i] - closes[i-1]), abs(lows[i] - closes[i-1]))
        plus_dm  = high_diff if high_diff > low_diff and high_diff > 0 else 0
        minus_dm = low_diff  if low_diff  > high_diff and low_diff  > 0 else 0
        ...

    # Wilder's smoothing → DI → DX → ADX(DX의 14일 평균)
    plus_di  = 100 * plus_di_smooth  / atr
    minus_di = 100 * minus_di_smooth / atr
    dx = 100 * abs(plus_di - minus_di) / (plus_di + minus_di)
    adx = (adx * (period - 1) + dx_list[i]) / period   # Wilder smoothing
```

- 표준 14일 ADX, Wilder's Smoothing 사용
- 최소 29일치 일봉 데이터 필요 (`period * 2 + 1`)

| > 35 | 극단적 공포 | (하드블록) | 매수 자체 차단 |

> 사용자 인식 "VIX는 -5%로 감점" 의 정확한 의미: **가중치는 5%, 점수는 0/음수만 가능 → 결과적으로 0 ~ -2.5% 감점**.

## 4. 종합 점수의 이론적 범위

각 부분 점수의 가중 기여도를 다 더하면:

| 부분 점수 | 가중치 | 점수 범위 | 가중 기여 범위 |
|---|---|---|---|
| rise_score | 0.25 | 0.2 ~ 1.0 | +0.050 ~ +0.250 |
| tech_score | 0.25 | 0.0 ~ 1.0 | +0.000 ~ +0.250 |
| sentiment_score | 0.20 | 0.0 ~ 1.0 | +0.000 ~ +0.200 |
| volume_score | 0.15 | -0.5 ~ +0.6 | -0.075 ~ +0.090 |
| adx_score | 0.10 | -0.3 ~ +0.4 | -0.030 ~ +0.040 |
| vix_score | 0.05 | -0.5 ~ +0.0 | -0.025 ~ +0.000 |
| **합계 범위** | | | **-0.130 ~ +0.830** |

- **이론적 최댓값**: ~0.830 (모든 부분 점수가 최대)
- **이론적 최솟값**: ~-0.130 (모든 부분 점수가 최소)
- **매수 통과선**: 0.300

> 실제 trade_records의 composite_score를 보면 0.30 ~ 0.62 범위가 대부분이라 위 이론값과 일치합니다.

## 5. 매도 점수 시스템 (참고)

매수와 달리 매도는 점수합산이 아닌 **신호 카운트** 방식입니다. (`stock_recommendation_service.py:790-998`)

### 매도 트리거 (OR 조건)

1. **ATR 기반 동적 익절/손절** (`trade_records.take_profit_price`/`stop_loss_price`)
   - 익절가 = `매수가 + ATR × 2.5`
   - 손절가 = `매수가 - ATR × 1.5`
   - ATR 정보 없으면 고정비율 폴백 (+5% 익절 / -7% 손절)

2. **기술적 매도 신호 카운트** (4개)
   - ① 데드 크로스 (`SMA20 < SMA50`)
   - ② RSI > 70 (과매수)
   - ③ MACD 매도 신호 (`MACD < Signal`)
   - ④ 패닉셀 (거래량 ≥ 2배 + 당일 ≤ -3% 하락)

3. **신호 임계치**
   - 신호 3개 이상 → 매도
   - 신호 2개 이상 + 감성<-0.15 → 매도
   - VIX > 30 + 신호 2개 이상 → 매도
   - VIX > 40 + 신호 1개 이상 → 매도

4. **ADX 보정**: ADX > 25이면 필요 신호 수 1개 차감 (강한 추세는 신뢰도 높음)

## 6. 실제 trade_records 사례로 검증

DB의 가장 최근 매수 기록(id=14, ADBE)을 예로 들면:

```
{
  "id": 14, "ticker": "ADBE",
  "buy_price": 240.53, "quantity": 178,
  "atr": null,                       ← ATR 계산 실패 케이스
  "take_profit_price": null,         ← ★ 안전장치 누락
  "stop_loss_price": null,           ← ★ 안전장치 누락
  "composite_score": 0.4581
}
```

composite_score 0.4581이 어떻게 나왔는지는 매수 시점 데이터(rise_probability, RSI, MACD, sentiment, volume_ratio, adx, vix)가 있어야 역산 가능합니다.
trade_records 테이블에는 부분 점수가 저장되지 않으므로 (`composite_score` 만 저장), 추후 디버깅을 위해 **부분 점수도 같이 저장하는 컬럼 추가를 검토할 만**합니다.
