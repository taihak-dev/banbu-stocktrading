## 1. 문제의 본질 — Wall Street 진단

### 1-1. "Sum of Weighted Scores" 의 함정

학계와 업계의 합의: **단순 가중합 멀티팩터는 가장 큰 분산을 가진 팩터가 결정을 지배함.**

수식으로 보면:

```
composite_score = Σ wᵢ · fᵢ

Var(composite_score) = Σ wᵢ² · Var(fᵢ) + 2·Σ wᵢ·wⱼ·Cov(fᵢ, fⱼ)
```

우리 시스템에서:
- **Var(rise_score) ≈ 0.06** (5단계 0.2~1.0 분포)
- Var(tech_score) ≈ 0.005 (사전필터로 0.71~1.0 압축)
- Var(sentiment_score) ≈ 0.01 (실측 0.35~0.65 분포)
- Var(volume_score) ≈ 0.003 (평일 0~0.3 위주)
- Var(adx_score) ≈ 0.04
- Var(vix_score) ≈ 0 (종목간 동일)

→ **rise_score 분산이 다른 팩터 합보다 큼 → 사실상 단일 팩터 시스템**

### 1-2. AQR Capital, Two Sigma, Renaissance 가 쓰는 표준 해법

월스트리트 정통 퀀트의 기본기:

1. **Cross-sectional standardization** — 매일 후보군 내에서 각 팩터를 z-score 변환하여 분산 동등화
2. **Information Coefficient (IC) tracking** — 팩터별 예측력을 rolling window 로 측정하여 검증
3. **Risk Parity** — 팩터별로 동등한 위험 기여도 부여
4. **Multi-strategy combining** — 모멘텀 + 역추세 + 퀄리티 등 상관관계 낮은 전략 결합
5. **Regime-aware allocation** — VIX/yield curve/breadth 로 시장 regime 분류 후 전략 비중 조정

우리 시스템은 **위 5가지 중 하나도 적용 안 됨** → 진정한 멀티팩터 시스템이라 부르기 어려움.

## 2. 개선 설계 — 3-Layer 아키텍처

### Layer 1: Cross-Sectional Z-Score 정규화 (P0, 즉시 적용)

#### 핵심 아이디어

매일 후보 N개에 대해 각 팩터를 **그날의 N개 안에서 z-score 로 표준화**.

```
z(rise_score)ᵢ = (rise_scoreᵢ - mean(rise_score)) / std(rise_score)
```

**효과**:
- 모든 팩터가 평균 0, 표준편차 1 로 정규화됨
- 명목 가중치가 곧 실효 가중치가 됨
- 25% 라고 하면 실제로 25% 영향력 발휘

#### 코드 설계

```python
# app/services/scoring_service.py (신규)
import numpy as np

def cross_sectional_zscore(values: list[float]) -> list[float]:
    """후보군 내 z-score 정규화. NaN 은 0 (평균) 처리."""
    arr = np.array([v if v is not None else np.nan for v in values])
    mean = np.nanmean(arr)
    std = np.nanstd(arr)
    if std < 1e-9:  # 모든 값 동일
        return [0.0] * len(values)
    z = (arr - mean) / std
    z = np.nan_to_num(z, nan=0.0)
    # winsorize at ±3 (극단치 제거)
    z = np.clip(z, -3.0, 3.0)
    return z.tolist()


def compute_composite_v2(candidates: list[dict], vix_value: float) -> list[dict]:
    """
    개선된 composite_score 계산.
    각 팩터를 cross-sectional z-score 변환 후 가중합.
    """
    n = len(candidates)
    if n == 0:
        return []

    # 팩터별 raw 값 추출
    rise_raw = [c.get("rise_probability", 0) for c in candidates]
    rsi_raw = [c.get("rsi", 50) for c in candidates]
    macd_diff_raw = [c.get("macd", 0) - c.get("signal", 0) for c in candidates]
    sma_diff_raw = [(c.get("sma20", 0) / c.get("sma50", 1) - 1) for c in candidates]
    sentiment_raw = [c.get("sentiment_score", 0) or 0 for c in candidates]
    volume_raw = [c.get("volume_ratio", 1) or 1 for c in candidates]
    adx_raw = [c.get("adx", 20) or 20 for c in candidates]

    # 각 팩터를 z-score 변환
    z_rise = cross_sectional_zscore(rise_raw)
    z_rsi = cross_sectional_zscore(rsi_raw)         # 단, RSI 는 별도 처리 (mean reversion)
    z_macd = cross_sectional_zscore(macd_diff_raw)
    z_sma = cross_sectional_zscore(sma_diff_raw)
    z_sent = cross_sectional_zscore(sentiment_raw)
    z_vol = cross_sectional_zscore(volume_raw)
    z_adx = cross_sectional_zscore(adx_raw)

    # 기술 합성 (3개 z-score 평균)
    z_tech = [(z_macd[i] + z_sma[i] + min(z_rsi[i], -z_rsi[i] + 2)) / 3
              for i in range(n)]

    # VIX 는 시장 메트릭이므로 absolute 처리
    vix_factor = vix_score_absolute(vix_value)  # -0.5 ~ 0

    results = []
    for i, c in enumerate(candidates):
        # 가중합 (모두 z-score 기준이므로 가중치가 곧 실효 영향력)
        composite = (
            0.30 * z_rise[i]    # ML
            + 0.25 * z_tech[i]   # 기술 (3 sub-factor 평균)
            + 0.15 * z_sent[i]   # 감성
            + 0.12 * z_vol[i]    # 거래량
            + 0.08 * z_adx[i]    # 추세 강도
            + 0.10 * vix_factor  # VIX (cross-sectional 효과 없으므로 작은 비중)
        )
        c["composite_v2"] = composite
        c["z_factors"] = {
            "rise": z_rise[i], "tech": z_tech[i], "sent": z_sent[i],
            "vol": z_vol[i], "adx": z_adx[i], "vix": vix_factor,
        }
        results.append(c)

    return results
```

#### 통과 기준 변경

기존: `composite_score >= 0.3`
신규: `composite_v2 >= 0.5` (z-score 기반이라 0이 평균, 0.5 = 후보 평균보다 0.5σ 이상)

#### 효과 검증

```
Before (raw):
  rise_score: 변동폭 0.20 → 가중 후 0.05~0.25 (실효 ~85% 변별력)
  vix_score:  변동폭 0    → 가중 후 0       (실효 0% 변별력)

After (z-score):
  z_rise:  변동폭 ~2σ      → 가중 30% 적용
  z_vix:   변동폭 0 (시장메트릭)  → 가중 10% 다른 팩터에 재배분
  → 진정한 30% / 25% / 15% / 12% / 8% / 10% 적용
```

### Layer 2: Multi-Track 전략 시스템 (P1, 1개월)

#### 핵심 아이디어

**하나의 점수 모델이 모든 시장 상황을 다루려는 환상에서 벗어남**. 헤지펀드 표준 패턴: **상관관계 낮은 여러 트랙을 병렬 운영**.

```
┌──────────────────────────────────────────────────────┐
│  Final Portfolio (10 종목 슬롯)                       │
├──────────────────────────────────────────────────────┤
│  ┌─────────────────┬──────────────────┬────────────┐ │
│  │ Momentum Track  │ Mean Reversion   │ Quality    │ │
│  │ (5 슬롯, 50%)   │ Track (3, 30%)   │ (2, 20%)   │ │
│  │                 │                  │            │ │
│  │ - 골든크로스     │ - RSI < 30 +     │ - 낮은 변동│ │
│  │ - MACD 매수     │   ADX > 25       │ - 긍정 감성│ │
│  │ - 양봉 고점     │ - 볼린저 하단     │ - 견고한   │ │
│  │ - ML rise > 5%  │ - 5일 평균 회귀  │   재무     │ │
│  └─────────────────┴──────────────────┴────────────┘ │
└──────────────────────────────────────────────────────┘
```

#### 트랙 1: Momentum (현재 시스템 유지 + 개선)

- composite_v2 (Layer 1) 적용
- 5 슬롯 할당 (포트폴리오의 50%)
- 주요 팩터: rise_score, golden_cross, MACD 매수, ADX > 25, 거래량 surge

#### 트랙 2: Mean Reversion (신규)

```python
def score_mean_reversion(candidate) -> float:
    """역추세 매수 점수.
    조건: 강한 추세 종목이 일시적으로 과매도 → 반등 매수
    """
    rsi = candidate["rsi"]
    adx = candidate.get("adx", 0)
    sma20 = candidate["sma20"]
    last_price = candidate["last_price"]
    bollinger_lower = sma20 - 2 * candidate.get("price_std_20", 0)

    # 핵심 조건
    is_oversold = rsi < 30
    has_strong_trend = adx > 25  # 강한 트렌드 종목만 (휴지통 종목 회피)
    near_bollinger_lower = last_price < bollinger_lower * 1.02
    sentiment_not_negative = (candidate.get("sentiment_score", 0) or 0) > -0.15

    score = 0.0
    if is_oversold:               score += 0.35
    if has_strong_trend:          score += 0.25
    if near_bollinger_lower:      score += 0.20
    if sentiment_not_negative:    score += 0.20

    return score  # 0 ~ 1.0
```

- 3 슬롯 할당 (포트폴리오의 30%)
- 횡보·약세장에서 우월한 성과
- 트랙 1 과 **상관계수 < 0.3** 목표 (분산 효과)

#### 트랙 3: Quality / Defensive (신규)

```python
def score_quality(candidate) -> float:
    """방어적 우량주 점수.
    조건: 낮은 변동성 + 안정적 펀더멘털 + 긍정 감성
    """
    atr_pct = candidate.get("atr", 0) / candidate.get("last_price", 1) * 100
    sentiment = candidate.get("sentiment_score", 0) or 0

    score = 0.0
    if atr_pct < 1.5:               score += 0.30  # 낮은 변동성
    if sentiment > 0.15:            score += 0.30  # 긍정 감성
    if candidate.get("rise_probability", 0) > 0:  score += 0.20  # 양수 ML
    if 40 < candidate["rsi"] < 60:  score += 0.20  # 중립 RSI

    return score
```

- 2 슬롯 할당 (포트폴리오의 20%)
- 약세장 + 고변동성 시기에 자본 보존
- TLT, GLD 같은 ETF 도 후보로 확장 가능

### Layer 3: Regime-Adaptive 가중치 (P2, 3개월)

#### 핵심 아이디어

**하나의 가중치 세트로 강세장/약세장/횡보장 모두 대응 불가**. Regime 분류 후 트랙 비중을 동적 조정.

#### Regime 분류 기준

```python
def classify_market_regime(vix: float, breadth: float, trend: float) -> str:
    """
    Args:
        vix: VIX 지수
        breadth: S&P 500 종목 중 50일선 위 비율 (%)
        trend: SPY 의 50일선 vs 200일선 비율
    Returns: "BULL" | "NEUTRAL" | "BEAR" | "PANIC"
    """
    if vix > 35:
        return "PANIC"
    if vix < 15 and breadth > 65 and trend > 1.02:
        return "BULL"
    if vix > 25 or breadth < 35 or trend < 0.98:
        return "BEAR"
    return "NEUTRAL"
```

#### Regime 별 트랙 비중

| Regime | Momentum | Mean Reversion | Quality | 매수 활성화 |
|---|---|---|---|---|
| BULL (VIX<15, breadth>65) | **70%** | 20% | 10% | 100% |
| NEUTRAL (default) | 50% | 30% | 20% | 100% |
| BEAR (VIX>25 or breadth<35) | 20% | 30% | **50%** | 50% (현금 50% 유지) |
| PANIC (VIX>35) | 0% | 0% | 0% | **0% (전면 중단, 기존 매도만)** |

## 3. Information Coefficient (IC) 모니터링

### 핵심 아이디어

**가중치를 한 번 정하고 끝내지 않고, 각 팩터의 예측력을 지속 측정**해서 약해진 팩터의 가중치를 자동으로 줄임.

### IC 정의

```
IC(factor) = Spearman_correlation(factor_value_t, return_{t→t+14d})
```

각 팩터 점수와 14일 후 실제 수익률의 순위 상관계수. **IC > 0.05** 이면 알파 있는 팩터로 간주 (업계 표준).

### 측정 + 자동 재가중

```python
# app/services/ic_monitor_service.py
def compute_rolling_ic(factor_name: str, window_days: int = 60) -> float:
    """과거 60일 데이터에서 팩터의 IC 계산"""
    from scipy.stats import spearmanr

    # trade_records + stock_recommendations 조인해서 데이터 수집
    history = fetch_factor_history(factor_name, window_days)

    factor_values = [h["factor_value"] for h in history]
    realized_returns = [h["return_14d"] for h in history]

    ic, _ = spearmanr(factor_values, realized_returns)
    return ic if not np.isnan(ic) else 0.0


def adaptive_weights() -> dict:
    """IC 기반으로 가중치 자동 조정 (월 1회 실행)"""
    factors = ["rise", "tech", "sent", "vol", "adx"]
    ics = {f: compute_rolling_ic(f) for f in factors}

    # Negative IC 팩터는 0 가중치 (예측 못 하는 팩터 배제)
    positive_ics = {f: max(ic, 0) for f, ic in ics.items()}

    total = sum(positive_ics.values())
    if total < 1e-9:
        return DEFAULT_WEIGHTS

    return {f: ic / total for f, ic in positive_ics.items()}
```

## 6. 위험 + 운영 가이드

### 6-1. 새 시스템 도입 시 주의사항

**a. Look-ahead bias 방지**
- z-score 계산 시 **그날 시장 마감 후 데이터 사용 금지**
- backtest 와 라이브 트레이딩의 데이터 시점 동일하게 유지

**b. Regime 전환 시 jitter**
- VIX 가 정확히 25 근처를 왔다갔다할 때 BEAR ↔ NEUTRAL 빈번 전환 → hysteresis 적용
- BULL→NEUTRAL: VIX 진입 임계값 17, 이탈 임계값 14

**c. Capacity 한계**
- 거래량 적은 종목을 z-score 가 추천할 수 있음
- 일평균 거래대금 $100M 미만은 universe 에서 제외

**d. Track 간 동일 종목 중복 매수 방지**
- 같은 종목이 momentum + mean_reversion 둘 다 통과 가능
- 최종 슬롯 할당 시 dedup 필요

### 6-2. 모니터링 대시보드

- 일일: 트랙별 매수 종목 수 + composite_v2 분포 (히스토그램)
- 주간: 트랙별 PnL + Sharpe + 상관계수 매트릭스
- 월간: IC 리포트 + 가중치 자동 조정 결과

### 6-3. 롤백 플랜

문제 발견 시 환경변수 한 줄로 v1 으로 즉시 복귀:

```python
# app/core/config.py
USE_SCORING_V2: bool = os.getenv("USE_SCORING_V2", "false").lower() == "true"

# app/services/stock_recommendation_service.py
if settings.USE_SCORING_V2:
    candidates = compute_composite_v2(candidates, vix_value)
else:
    candidates = compute_composite_v1(candidates, vix_value)  # 기존
```

## 8. Wall Street 자기 점검 (이 기획의 한계)

20년 PM 으로서 솔직하게 짚어두는 약점들:

### 8-1. 과적합 위험

- Multi-track + regime adaptive 는 자유도가 매우 높음
- 백테스트 데이터에 over-fit 될 위험 → out-of-sample 검증 필수

### 8-2. 자본 capacity 한계

- 현재 모의계좌 약 ₩200M 규모
- 이 규모에서는 시장 임팩트 무시 가능
- 그러나 자본 1B+ 로 확장 시 거래량 ≥ $50M 종목으로 universe 축소 필요

### 8-3. ML rise_score 자체의 한계

- 14일 forecast horizon 은 너무 짧음 (단기 노이즈 큼)
- 90일 lookback 은 regime change 못 따라감
- ML 모델 자체의 walk-forward retraining 빈도도 검토 필요

### 8-4. 인적 요소

- LLM "거부권" 은 좋지만 LLM 학습 컷오프 후 사건은 모름
- 시스템에 인간 risk officer 의 manual override 인터페이스 필요 (P3)
