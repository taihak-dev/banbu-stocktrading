# 매일 수동 작업 자동화 방안

> 매일 사람이 직접 돌리고 있는 4단계를 어떻게 무인 자동화할 수 있는지에 대한 분석과 현실적 해결책 정리.

---

## 1. 현재 수동 작업 (Daily Manual Workflow)

| # | 작업 | 위치 | 소요 시간 | 자동화 난이도 |
|---|---|---|---|---|
| 1 | 경제지표 + 주가 수집 | 시스템 시작 시 자동 (`update_economic_data_in_background`) | 1~2분 | 🟢 이미 부분 자동화 |
| 2 | **ML 예측 (Transformer 학습)** | **Google Colab 수동 실행** | **약 7분** | 🔴 가장 어려움 |
| 3 | 기술적 분석 + 점수 계산 | `POST /stocks/recommendations/recommended-stocks/generate-technical-recommendations` | 30초~1분 | 🟢 쉬움 |
| 4 | LLM 최종 검토 + 매수 주문 | `POST /llm/review-buy-candidates` | 10~30초 | 🟢 쉬움 |

### 1-1. 현재 흐름의 문제점

- **사람이 매일 매일 4단계를 순서대로 트리거** 해야 함
- Colab은 브라우저 의존적이라 **휴가/출장/잠 등으로 누락 위험**
- 단계 사이 의존성: 1 → 2 → 3 → 4 순으로 실행되어야 함 (앞 단계가 안 끝나면 뒤가 의미 없음)
- 매수 시점(뉴욕 10:30 ET = 한국 23:30 KST 또는 00:30 KST)에 맞춰야 해서 **시간 압박** 있음

### Step 2: ML 예측 (Colab) — 🔴 핵심 병목

#### 왜 어려운가?

| Colab의 한계 | 영향 |
|---|---|
| 브라우저 세션이 끊기면 멈춤 | 자동 스케줄링 불가 |
| Free 플랜은 12시간 후 자동 종료 | 장시간 작업 불안정 |
| 공식 cron/스케줄링 API 없음 | 외부 트리거 불가능 |
| Selenium 자동화는 brittle | TOS 위반 + 잦은 깨짐 |

#### 검토한 우회책 (전부 비추)

- ❌ **Colab Pro + Background Execution**: 백그라운드 실행은 되지만 **시간 트리거가 없음**
- ❌ **Selenium으로 Colab 클릭 자동화**: 약관 위반, 매번 깨짐
- ❌ **GitHub Actions**: GPU 없음 → 학습 불가능

#### ✅ 현실적 대안 (3가지)

##### 옵션 A: **Kaggle Notebooks** (무료, 추천 1순위)

| 항목 | 내용 |
|---|---|
| GPU | **무료 P100 / T4** (주당 30시간) |
| 스케줄링 | **공식 지원** (Notebook → Settings → Schedule) |
| 비용 | **₩0** |
| 마이그레이션 | `predict_colab.py` 거의 그대로 복붙 |
| 데이터 | 이미 Supabase로 저장하고 있어서 그대로 작동 |

**작업량**: Kaggle 계정 만들고 노트북 업로드 → "Schedule" 메뉴에서 매일 시간 지정. **1~2시간**.

##### 옵션 B: **Modal.com** (서버리스 GPU, 추천 2순위)

| 항목 | 내용 |
|---|---|
| GPU | A10G / T4 / L4 등 선택 가능 |
| 스케줄링 | `@modal.function(schedule=modal.Cron("30 1 * * *"))` 한 줄 |
| 비용 | **월 $30 무료 크레딧** (하루 7분 학습 → 월 사용량 약 $1~3) |
| 마이그레이션 | Python 함수로 래핑 필요 (1~2시간) |
| 통합성 | FastAPI에서 직접 호출 가능 (`modal.Function.from_name(...).remote()`) |

**장점**: 코드가 우리 시스템과 한몸이 됨. FastAPI에서 "지금 학습 돌려" 버튼도 만들 수 있음.

##### 옵션 C: **RunPod / Vast.ai** (GPU 시간 임대)

| 항목 | 내용 |
|---|---|
| GPU | RTX 3090 / 4090 등 |
| 스케줄링 | 직접 cron + API 시작/종료 |
| 비용 | RTX 3090 ~$0.20/hr → 7분/일 = **월 ₩1,000~2,000** |
| 마이그레이션 | Docker 이미지 만들고 startup 스크립트 작성 (반나절) |

**장점**: 가장 저렴. **단점**: 운영 복잡도 ↑

## 4. 단계별 구현 가이드

### Phase 1 (가장 시급, 1~2시간 작업): **API 3+4 자동화**

#### 4-1. `_run_generate_technical_recommendations` 추가

`app/utils/scheduler.py` 에 함수 추가:

```python
def _run_generate_technical_recommendations():
    """기술적 지표 + 감성분석 갱신 (매일 KST 22:50)"""
    try:
        logger.info("기술적 지표 생성 시작")
        from app.services.stock_recommendation_service import StockRecommendationService
        service = StockRecommendationService()
        tech_result = service.generate_technical_recommendations()
        logger.info(f"기술적 지표 생성 완료: {tech_result['message']}")

        logger.info("뉴스 감성 분석 시작")
        sentiment_result = service.fetch_and_store_sentiment_for_recommendations()
        logger.info(f"뉴스 감성 분석 완료: {sentiment_result['message']}")
        return True
    except Exception as e:
        logger.error(f"기술 지표 + 감성 갱신 실패: {e}", exc_info=True)
        return False
```

`start_scheduler()` 에 등록:

```python
schedule.every().day.at("22:50").do(_run_generate_technical_recommendations)
```

> Step 4(매수)는 이미 `_run_auto_buy()` 에서 LLM 검토까지 자동으로 함 → 추가 작업 없음.

### Phase 2 (가장 임팩트, 1~2시간 작업): **Kaggle Notebooks로 ML 이전**

#### 4-2. Kaggle 마이그레이션 단계

1. [Kaggle](https://kaggle.com) 가입
2. 우측 상단 → **Code → New Notebook**
3. `predict_colab.py` 내용 그대로 셀에 붙여넣기
4. 우측 패널에서 **Settings**:
   - **Accelerator**: `GPU T4 x2` 또는 `GPU P100`
   - **Internet**: ON (Supabase 호출 필요)
5. **Add-ons → Secrets** 에서 `SUPABASE_URL`, `SUPABASE_KEY` 추가하고 코드에서 `os.environ.get("SUPABASE_URL")` 로 읽도록 수정
6. **Save Version → Save & Run All (Commit)** 으로 한 번 정상 실행 확인
7. 우측 상단 ⋮ → **Schedule** → **Daily** → 시간 지정 (KST 22:30 = UTC 13:30)

> Kaggle 무료 GPU 주당 30시간 → 매일 7분 × 30일 = **월 약 3.5시간** → 여유 충분

#### 4-3. 코드 수정 (Supabase 키 환경변수화)

import modal

app = modal.App("stock-prediction")
image = modal.Image.debian_slim().pip_install("tensorflow", "supabase", "pandas", "scikit-learn")

@app.function(
    gpu="T4",
    image=image,
    schedule=modal.Cron("30 13 * * *"),  # UTC 13:30 = KST 22:30
    secrets=[modal.Secret.from_name("supabase-creds")],
    timeout=900,
)
def train_and_predict():
    # predict_colab.py 의 main() 그대로 호출

    main()
```

```

이후 FastAPI 라우트에서 수동 호출도 가능:

## 8. 주의사항 & 함정

### 8-1. 시간대 헷갈리지 마세요

| 일정 | UTC | KST | NY (EST/EDT) |
|---|---|---|---|
| 미국 장 시작 | 14:30 (EDT) | 23:30 | 09:30 |
| 매수 실행 | 14:30 (EDT) | 00:30 (다음날) | 10:30 |

→ Kaggle/Modal 스케줄링은 **UTC 기준** 입력해야 함. 서머타임 변경 시 1시간 차이 주의.

### 8-2. ML 학습 실패 시 폴백

만약 Kaggle/Modal에서 학습이 실패하면 → **stock_analysis_results 테이블이 어제 데이터 그대로** 남아있음.
시스템은 "어제 예측"으로 매수 시도 → 정확도 ↓
**대응**:
- `stock_analysis_results.created_at` 이 24시간 이상 오래됐으면 매수 차단하는 가드 추가
- `app/services/stock_recommendation_service.py:get_stock_recommendations()` 에 다음 조건 추가:

```python
from datetime import datetime, timedelta

latest = pd.to_datetime(df['created_at']).max()
if latest < datetime.now() - timedelta(hours=24):
    return {"message": "ML 예측 데이터가 24시간 이상 오래되어 매수 차단", "recommendations": []}
```

### 8-3. Kaggle 시크릿 로테이션

Supabase 키를 Kaggle Secret에 넣으면 키 변경 시 두 군데 갱신 필요. **공식 운영 환경에는 Modal이 더 깔끔**.

### 8-4. 알림(Slack/이메일) 추가 권장

자동화의 함정: **실패해도 모름**. 이래서 다음 중 하나 추가 권장:
- Kaggle 노트북 실패 시 이메일 (Kaggle 설정에서 가능)
- Modal 실패 시 Slack webhook 호출
- FastAPI에서 다음날 아침 "어제 매수 N건 / P&L 요약" 메시지 발송
