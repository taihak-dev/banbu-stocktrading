## 4. PART 1: 데이터 로드 + 전처리

### 4-1. Supabase에서 페이지네이션으로 전체 데이터 로드 (`predict_colab.py:86-97`)

```python
def get_all_data(table_name):
    all_data = []
    offset = 0
    limit = 1000
    while True:
        response = supabase.table(table_name).select("*").order("날짜", desc=False)\
                          .limit(limit).offset(offset).execute()
        data = response.data
        if not data:
            break
        all_data.extend(data)
        offset += limit
    return all_data
```

#### 왜 페이지네이션?

Supabase는 한 번에 최대 1000개 행만 반환합니다. 전체 데이터(약 5000+ 행)를 가져오려면 **1000행씩 끊어서 여러 번 요청** 해야 합니다.

| 호출 | offset | 결과 |
|---|---|---|
| 1회차 | 0 | 0~999번 행 |
| 2회차 | 1000 | 1000~1999번 행 |
| ... | ... | ... |
| n회차 | n*1000 | 빈 배열 → 루프 종료 |

### 4-2. 데이터 정제 (`predict_colab.py:100-125`)

```python
def get_stock_data_from_db():
    all_data = get_all_data("economic_and_stock_data")
    df = pd.DataFrame(all_data)

    df['날짜'] = pd.to_datetime(df['날짜'])     # 문자열 → 날짜
    df.sort_values(by='날짜', inplace=True)    # 날짜 오름차순
    df.reset_index(drop=True, inplace=True)

    df = df.ffill().bfill()                    # ★ 결측치 채우기

    exclude_columns = ['날짜', 'id']
    numeric_columns = [col for col in df.columns if col not in exclude_columns]
    df[numeric_columns] = df[numeric_columns].apply(pd.to_numeric, errors='coerce')

    nan_ratios = df[numeric_columns].isna().mean()
    valid_columns = [col for col in numeric_columns if nan_ratios[col] < 1.0]
    df.dropna(subset=valid_columns, inplace=True)
    df.reset_index(drop=True, inplace=True)
    return df
```

#### `ffill()` vs `bfill()` 차이

| 메서드 | 의미 | 예시 |
|---|---|---|
| `ffill` (forward fill) | **앞쪽 값으로 뒤를 채움** | [100, NaN, NaN, 105] → [100, 100, 100, 105] |
| `bfill` (backward fill) | **뒤쪽 값으로 앞을 채움** | [NaN, NaN, 100, 105] → [100, 100, 100, 105] |

> 주말/공휴일이라 가격이 없는 날 → ffill로 직전 거래일 가격 그대로 사용.
> 데이터 시작점에 NaN이 있을 때 → bfill로 미래 값으로 메꿈.

#### 왜 `nan_ratios < 1.0` 필터?

```python
nan_ratios = df[numeric_columns].isna().mean()  # 각 컬럼의 NaN 비율
valid_columns = [col for col in numeric_columns if nan_ratios[col] < 1.0]
```

특정 컬럼이 **100% NaN** (즉 ffill/bfill 후에도 값이 하나도 없는 경우) → 그 컬럼 자체가 망가진 것. 이런 컬럼은 dropna에서 제외하지 않으면 모든 행이 삭제됨.

### 4-3. 데이터 스케일링 (`predict_colab.py:300-306`)

```python
stock_scaler = MinMaxScaler()
econ_scaler = MinMaxScaler()

data_scaled = data.copy()
data_scaled[TARGET_COLUMNS] = stock_scaler.fit_transform(data[TARGET_COLUMNS])
data_scaled[ECONOMIC_FEATURES] = econ_scaler.fit_transform(data[ECONOMIC_FEATURES])
```

#### MinMaxScaler가 뭐죠?

각 컬럼을 **0과 1 사이로 정규화** 합니다.

```
정규화된 값 = (원본값 - 최솟값) / (최댓값 - 최솟값)
```

| 종목 | 원본 가격 | 최솟값 | 최댓값 | 정규화 후 |
|---|---|---|---|---|
| 애플 | 175 | 50 | 200 | 0.833 |
| 테슬라 | 250 | 100 | 400 | 0.500 |
| 펩시코 | 150 | 130 | 180 | 0.400 |

#### 왜 스케일링이 필요한가?

- **애플 ~$175** vs **테슬라 ~$250** vs **VIX 지수 ~20** vs **금리 ~5.5**
- 단위가 다 다르면 신경망이 큰 숫자(가격)를 더 중요하다고 잘못 학습함
- 모두 0~1로 통일하면 **공정한 비교** 가 됨

#### 두 개의 별도 스케일러를 쓰는 이유

```python
stock_scaler = MinMaxScaler()    # ← TARGET_COLUMNS 전용 (27개 종목)
econ_scaler = MinMaxScaler()      # ← ECONOMIC_FEATURES 전용 (37개 지표)
```

예측 후 **역변환** (`inverse_transform`) 할 때, 어떤 스케일러로 변환했는지 정확히 매칭해야 원래 가격으로 복원 가능합니다. 그래서 종목용/지표용 스케일러를 분리합니다.

### 5-1. 학습 데이터 생성 (`predict_colab.py:316-328`)

```python
LOOKBACK = 90              # 과거 90일 보고
FORECAST_HORIZON = 14      # 14일 후 예측

X_stock_train = []
X_econ_train = []
y_train = []

for i in range(LOOKBACK, len(data_scaled) - FORECAST_HORIZON):
    X_stock_train.append(data_scaled[TARGET_COLUMNS].iloc[i - LOOKBACK:i].values)
    X_econ_train.append(data_scaled[ECONOMIC_FEATURES].iloc[i - LOOKBACK:i].values)
```

#### 슬라이딩 윈도우 비유

```
전체 데이터:  [Day0, Day1, Day2, ..., Day1000]

i = 90  → 입력: Day0~Day89   (90일치)  →  정답: Day103 가격 (90+14-1)
i = 91  → 입력: Day1~Day90              →  정답: Day104 가격
i = 92  → 입력: Day2~Day91              →  정답: Day105 가격
...
i = 985 → 입력: Day895~Day984           →  정답: Day998 가격
```

> 이렇게 만들면 **1주가지정 데이터셋에서 약 900~1000개의 학습 샘플**이 생깁니다.

#### 입력은 두 종류 (멀티 모달)

| 입력 | 모양(shape) | 의미 |
|---|---|---|
| `X_stock_train` | `(샘플수, 90, 27)` | 90일치 × 27개 종목 가격 |
| `X_econ_train` | `(샘플수, 90, 37)` | 90일치 × 37개 경제 지표 |
| `y_train` | `(샘플수, 27)` | 14일 후 27개 종목의 정답 가격 |

### 5-2. Transformer 모델 구조 (`predict_colab.py:130-163`)

```python
def transformer_encoder(inputs, num_heads, ff_dim, dropout=0.1):
    # 1) Multi-Head Self-Attention
    attention_output = MultiHeadAttention(num_heads=num_heads, key_dim=inputs.shape[-1])(inputs, inputs)
    attention_output = Dropout(dropout)(attention_output)
    attention_output = Add()([inputs, attention_output])              # ← Residual
    attention_output = LayerNormalization(epsilon=1e-6)(attention_output)

    # 2) Feed-Forward Network
    ffn = Dense(ff_dim, activation="relu")(attention_output)
    ffn = Dense(inputs.shape[-1])(ffn)
    ffn_output = Dropout(dropout)(ffn)
    ffn_output = Add()([attention_output, ffn_output])               # ← Residual
    ffn_output = LayerNormalization(epsilon=1e-6)(ffn_output)
    return ffn_output
```

#### Transformer를 쉽게 비유하면

**"문장 속 단어들이 서로를 바라보면서 누가 누구와 관련 있는지 파악하는 구조"**

여기서는 단어 대신 **시간(90일)** 이 들어감.
> "60일 전의 가격이 오늘의 가격에 얼마나 영향을 줬는지" 같은 시계열 관계를 모델이 스스로 학습.

#### 핵심 구성 요소

| 레이어 | 역할 | 비유 |
|---|---|---|
| **MultiHeadAttention** | 시점 간 관계 학습 | 8명의 분석가가 각자 다른 관점으로 데이터 분석 |
| **Dropout** | 일부 뉴런 무작위 차단 | 학습 시 의존성 줄여 과적합 방지 |
| **Add (Residual)** | 입력값 그대로 더하기 | 정보 손실 방지, 깊은 신경망에서 학습 안정화 |
| **LayerNormalization** | 값 정규화 | 학습 안정화 |
| **Dense (Feed-Forward)** | 비선형 변환 | 특징 추출 |

### 5-3. 두 입력 통합 모델 (`predict_colab.py:144-163`)

```python
def build_transformer_with_two_inputs(stock_shape, econ_shape, num_heads, ff_dim, target_size):
    # 입력 1: 주식 가격
    stock_inputs = Input(shape=stock_shape)
    stock_encoded = stock_inputs
    for _ in range(4):                                          # ← 4번 쌓음
        stock_encoded = transformer_encoder(stock_encoded, num_heads=num_heads, ff_dim=ff_dim)
    stock_encoded = Dense(64, activation="relu")(stock_encoded)

    # 입력 2: 경제 지표
    econ_inputs = Input(shape=econ_shape)
    econ_encoded = econ_inputs
    for _ in range(4):
        econ_encoded = transformer_encoder(econ_encoded, num_heads=num_heads, ff_dim=ff_dim)
    econ_encoded = Dense(64, activation="relu")(econ_encoded)

    # 두 입력 결합
    merged = Add()([stock_encoded, econ_encoded])               # ← 두 흐름 합침
    merged = Dense(128, activation="relu")(merged)
    merged = Dropout(0.2)(merged)
    merged = GlobalAveragePooling1D()(merged)                   # ← 시간축 평균 (90→1)
    outputs = Dense(target_size)(merged)                        # ← 27개 종목 출력

    return Model(inputs=[stock_inputs, econ_inputs], outputs=outputs)
```

#### 모델 구조 시각화

```
주식 가격 (90일×27종목)               경제지표 (90일×37지표)
        │                                    │
        ▼                                    ▼
┌───────────────────┐               ┌───────────────────┐
│ Transformer Enc x 4│               │ Transformer Enc x 4│
└───────────────────┘               └───────────────────┘
        │                                    │
        ▼                                    ▼
   Dense(64, relu)                      Dense(64, relu)
        │                                    │
        └────────────► Add ◄────────────────┘   (두 흐름 합치기)
                        │
                        ▼
                  Dense(128, relu)
                        │
                        ▼
                  Dropout(0.2)
                        │
                        ▼
              GlobalAveragePooling1D    (시간축 90 → 1로 압축)
                        │
                        ▼
                 Dense(27)              (27개 종목 14일 후 가격)
```

#### 왜 4겹씩 쌓나?

- 1겹: 단순 패턴 (어제/오늘 관계)
- 2겹: 중기 패턴 (1주 트렌드)
- 3겹: 복합 패턴 (월간 사이클)
- 4겹: 고차 패턴 (분기 이상의 흐름)

깊이가 깊을수록 **추상적인 관계** 를 학습할 수 있지만, 너무 깊으면 과적합 위험 + 학습 시간 증가.

### 5-4. 학습 (`predict_colab.py:343-348`)

```python
model.compile(optimizer=Adam(learning_rate=LEARNING_RATE), loss='mse', metrics=['mae'])

history = model.fit(
    [X_stock_train, X_econ_train], y_train,
    epochs=EPOCHS,

```

#### 손실 함수 (loss): MSE

```
MSE = 평균( (예측값 - 실제값)² )
```

큰 오차에 더 큰 페널티 → 큰 실수를 줄이는 방향으로 학습.

#### 옵티마이저: Adam

학습률을 자동으로 조정해주는 똑똑한 옵티마이저. 거의 모든 딥러닝에서 기본으로 사용.

#### Epoch이 뭐죠?

학습 데이터 전체를 한 번 도는 단위.
- 데이터 1000개, batch_size 32 → 1 epoch = 32개씩 묶어 약 32번 학습
- 50 epochs → 같은 데이터를 50번 반복 학습

> 학습이 진행되면서 loss가 점점 작아져야 정상. 진행 그래프는 `plt.plot(history.history['loss'])` 로 확인.

#### `inverse_transform` (스케일 역변환)

```python
predicted_prices_actual = stock_scaler.inverse_transform(predicted_prices)
```

모델 출력값은 0~1 범위인데 → 실제 달러 가격으로 복원하려면 학습 시 사용한 스케일러로 역변환.

```
실제 가격 = 정규화 값 × (최댓값 - 최솟값) + 최솟값
```

```python
# 미래 예측 부분은 정답이 없으므로 NaN으로 패딩
if actual_full.shape[0] < pred_len:
    nan_padding = np.full((pred_len - actual_full.shape[0], len(TARGET_COLUMNS)), np.nan)
    actual_full = np.vstack([actual_full, nan_padding])

# 종목별 컬럼 생성
result_data = pd.DataFrame({'날짜': today_dates})
for idx, col in enumerate(TARGET_COLUMNS):
    result_data[f'{col}_Predicted'] = predicted_prices_actual[:, idx]
    result_data[f'{col}_Actual'] = actual_full[:, idx]
```

### 7-1. 정확도 평가 (`predict_colab.py:168-202`)

```python
def evaluate_predictions(data, target_columns, forecast_horizon):
    metrics = []
    for col in target_columns:
        predicted = data[f'{col}_Predicted']
        actual = data[f'{col}_Actual'].shift(-forecast_horizon)  # ← 14일 미래로 시프트

        # NaN 제거
        valid_idx = ~predicted.isna() & ~actual.isna()
        predicted = predicted[valid_idx]
        actual = actual[valid_idx]

        mae = mean_absolute_error(actual, predicted)
        mse = mean_squared_error(actual, predicted)
        rmse = mse ** 0.5
        mape = (abs((actual - predicted) / actual).mean()) * 100
        accuracy = 100 - mape
        ...
```

#### 핵심 트릭: `shift(-forecast_horizon)`

```
[Day0_Predicted, Day1_Predicted, ..., Day100_Predicted]
        ↕ 비교해야 함
[Day14_Actual, Day15_Actual, ..., Day114_Actual]
```

`Day0_Predicted` 는 사실 "Day0 시점에서 14일 후를 예측한 값" → 이걸 `Day14_Actual` 과 비교해야 정확.
`shift(-14)` 가 정답 시리즈를 **14칸 앞으로 당겨서** 같은 인덱스로 비교 가능하게 만듦.

#### 평가 지표 4종

| 지표 | 산식 | 의미 |
|---|---|---|
| **MAE** | mean(\|예측-실제\|) | 평균 오차 절댓값 (단위: 달러) |
| **MSE** | mean((예측-실제)²) | 큰 오차에 페널티 |
| **RMSE** | √MSE | MAE처럼 달러 단위지만 큰 오차 가중 |
| **MAPE** | mean(\|예측-실제\|/실제) × 100 | 비율 오차 (%) |
| **Accuracy** | 100 - MAPE | "정확도" 명목 (사실 100-MAPE) |

> ⚠️ 여기서 "Accuracy"는 머신러닝의 진짜 분류 정확도가 아니라 단순히 **`100 - MAPE`** 계산값. 회귀 문제에서 쓰는 비공식 지표라는 점 주의.


## 13. 마지막 정리: 이 모델의 한계

이 모델은 **"과거 90일 패턴 → 14일 후 가격"** 만 학습합니다. 그러나 주가는:

1. **외부 충격 (블랙스완)**: COVID, 전쟁, 금리 발표 → 학습 데이터에 없으면 예측 불가
2. **개별 호재/악재**: 실적 발표, CEO 교체, 소송 → 모델은 가격/지표만 보지 뉴스 모름 → AlphaVantage 감성으로 보완
3. **시장 체제 변화**: 강세장 → 약세장 전환기에 정확도 급락
4. **종목별 편차**: 변동성 큰 종목(TSLA)은 정확도 낮고, 안정적 종목(COST)은 높음

이래서 시스템은 **ML 예측 단독으로 결정하지 않고**, 기술적 지표·감성·거래량·VIX·LLM 검토를 모두 결합해서 최종 매수를 결정합니다.
